## 总评

**B-（综合 ≈6.2/10）**——数据资产已达"个人版 CRSP"水准、研究方法论超出多数个人栈，
但运维是业余水平的定时炸弹。距专业生产级差三件事：

1. **灾备**：评估时备份定时器处于停用状态（7/5 起，已于本次评估当晚修复）、
   所有副本同盘、恢复从未演练；
2. **非价格域的幸存者偏差**：退市公司基本面覆盖 0.9%、13F 1.8%、时点市值只有
   2 年——"20 年无偏回测"目前只对价格类因子成立；
3. **自动化质量门**：健康报告无人调度、新闻停更 53 天无人察觉、30 天 17% 任务
   失败且诊断只剩 exit=1——平台还不能自己发现自己在腐烂。

## 镇库之宝（对个人研究者真正稀有的资产）

1. **无幸存者偏差的 20 年日线面板**：31.6M 行、18,037 只 CS/ETF（41.6% 已退市）、
   退市价格覆盖 94.9%/98.1%——"平民版 CRSP"（商业等价物数万美元/年），全栈最难复制。
2. **端到端可复现复权链**：335,897 vendor 事件 → 303,741 内部因子，9/9 著名拆股
   因子精确等于拆股比、跨 ex-date 价格分币级连续，2003 信任下限是文档化边界。
3. **5.06B 行分钟线归档**（2003-09~2026-04-23、17,006 只含退市）：散户几乎不存在的
   日内研究期权；注意在贬值（无实时源）。
4. **专业级评估卫生**：Newey-West IC t 值、扣成本分位 Sharpe、DTB3 超额、
   trials.parquet 按 git sha/参数/宇宙哈希登记每次试验——很多小基金都没有。
5. **独立的 SEC 免费腿**：12.9M 重述感知 PIT XBRL + 114M 行 13F + 5.36M 内部人——
   不依赖 Massive key，且正是修幸存者偏差的原料库。

## 硬道理（评估原话）

1. 你（评估时点）没有备份，且所有副本同在一块跑着 Plex 和游戏的盘上——失去平台最可能的
   方式不是量化 bug，是一块硬盘。分钟数据唯一后路在笔记本里，那不叫备份叫运气。
2. "20 年无幸存者偏差回测"只对价格因子成立——死掉的 6,059 家公司里 0.9% 有基本面，
   跑出的 20 年 Sharpe 是上界不是估计值，且研究层不会强迫你意识到。
3. 有些缺口是买回来的不是写代码写回来的——免费档天花板（做空 2024-05 前、13F 2013Q2 前、
   分钟无源）应该是有备忘录的显式决策，现在是每天计息的默认拖延。
4. 平台不会自己发现自己在腐烂——新闻 53 天、分钟 2.5 个月、17% 失败率 exit=1，
   靠的是你的注意力，而注意力会耗尽。
5. 已知脏数据躺了一年没修（NVDA high 半天就能修）——问题不在难度，在优先级习惯。

## 评估当晚的即时处置与 owner 决策（2026-07-06）

- 评估发现备份 timer 自 7/5 起停用。当晚曾重启并完成一次 9.7G 全新备份
  （含 20 年公司行动、因子、vwap 回填后的完整状态）；restore 演练进行中
  发现 stdin 流式恢复的限制（custom 格式不可 seek，需 docker cp 进容器）。
- **Owner 决策：取消每日备份系统（磁盘约束）**。timer 已再次停用并按指示
  清理旧 dump；保留唯一一份 2026-07-06 全量快照
  `/home/wenruifeng/stock_backups/postgres/stock_20260706T013317Z.dump`（9.7G）。
  当前灾备状态 = 这一份同盘快照 + Mac 上的源归档（day-aggs tgz / WithVW parquet /
  fundamentals tgz，可全量重放）+ SEC 免费数据可重拉。
- 含义（记录，不评判）：PG 灾备是"重放式"而非"快照式"——磁盘故障后恢复路径为
  重跑各导入管线（数天级 RTO），期间 Massive 免费档 730 天窗口外的 live 增量
  （做空数据等）不可重放。若未来加盘/云存储，重启 timer 只需
  `systemctl enable --now stock-postgres-backup.timer`（保留 7 天 override 已在位）。
- 差距地图中"灾备缺位"一项据此调整为 owner 已知悉并接受的风险，
  解除条件 = 存储扩容。其余项进路线图不变。
