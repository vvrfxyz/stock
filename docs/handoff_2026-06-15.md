```
        ├── size.py          # SizeFactor (log market cap)
        └── earnings_yield.py # EarningsYieldFactor (NI_TTM / mcap)
```

### 关键约定 (新模块)

1. **`event_table_to_asof_panel`** 是所有"事件表 → PIT 宽表"的单一真源。未来 insider / 13F / short / 股本因子都应复用 (`from research.factors.asof import event_table_to_asof_panel`)。
2. **wrapper 层 `_to_ns` 责任**: 通用工具不 coerce 时间列 dtype;包装层 (fundamentals / market_cap) 必须保留 `_to_ns` 调用。详见 `docs/codex_tasks/factors_asof_generic.md` "wrapper 层 dtype 归一化责任"。
3. **Factor 协议**: `name: ClassVar[str]` (不能 dataclass 实例字段),`Factor` 必须 `@runtime_checkable`,builtin 模块顶层 `register(SizeFactor())`。
4. **`FactorContext.as_of`** v1 仅元数据,builtin compute **不传给 loader**。loader 用 `dates + visible_delay_days` 推 PIT 边界。Spec 反需求 #10。
5. **trials.parquet long-form schema**: 一个 trial 多行 (每行一个 metric × horizon)。schema 加列要 bump `TRIALS_SCHEMA_VERSION`,**禁止 delete/rename 列**。
6. **`research/output/`** 在 `.gitignore`,trials.parquet 本地积累,跨机器搬运用 csv 导出。

### 已知约束 (spec 显式选择)

- **as_of 固定 = end (v1)**: 用 `pit_regression_max_abs_diff` 抽 60/120/180 交易日重算因子比较兜底。fundamentals.asof_panel 加 vintage as_of 入参后才能升级到滚动 as_of。
- **行业中性化未做**: spec 反需求明文留给 #8。当前 IC/IR 不区分 size/value 与行业暴露。
- **分位再平衡严格 = horizon 交易日** (`index[::h]`),不用 `rebalance_dates('M')`。
- **rank-IC 只报 Spearman**,不报 Pearson level-IC。
- **`run_baselines.py` 与 `evaluate.py` 禁止互相 import**,共享层仅限 `research.data + research.backtest + research.factors.protocol`。
- **无风险利率 rf=0**: Sharpe 虚高 0.3-0.5,所有评测建立在错基准上。**Next 期 #3 优先修**。
